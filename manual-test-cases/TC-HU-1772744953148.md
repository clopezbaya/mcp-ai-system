# Casos de Prueba Generados

## TC-001: Inicio de sesión exitoso con credenciales válidas
**Prioridad:** Alta

**Precondiciones:**
El usuario debe estar registrado previamente en el sistema. El navegador debe estar en la vista 'login_screen'.

**Pasos:**
1. Localizar el campo de texto con el selector #user-id.
2. Ingresar un nombre de usuario válido.
3. Localizar el campo de contraseña con el selector #password-id.
4. Ingresar la contraseña correspondiente.
5. Hacer clic en el botón con el selector #submit-btn.

**Datos:** `usuario: 'cliente_demo', contraseA: 'Pass1234$'`

**Resultado Esperado:**
El sistema autentica al usuario y redirige automáticamente al Dashboard del perfil de cliente.

---

## TC-002: Validación de error con credenciales incorrectas
**Prioridad:** Alta

**Precondiciones:**
El navegador debe estar en la vista 'login_screen'.

**Pasos:**
1. Ingresar un nombre de usuario no registrado o incorrecto en #user-id.
2. Ingresar cualquier cadena en #password-id.
3. Hacer clic en el botón con el selector #submit-btn.

**Datos:** `usuario: 'usuario_invalido', contraseA: 'error_pass'`

**Resultado Esperado:**
El sistema no permite el acceso y muestra un mensaje de error indicando que las credenciales fallaron.

---

## TC-003: Validación de campos obligatorios vacíos
**Prioridad:** Media

**Precondiciones:**
El navegador debe estar en la vista 'login_screen'.

**Pasos:**
1. Dejar vacío el campo #user-id.
2. Dejar vacío el campo #password-id.
3. Hacer clic en el botón con el selector #submit-btn.

**Datos:** `N/A`

**Resultado Esperado:**
El sistema impide el envío del formulario y muestra alertas visuales indicando que los campos son obligatorios.

---
